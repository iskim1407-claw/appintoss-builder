## 📊 1. 경쟁 분석 — 우리가 가진 것과 못 가진 것

### 글로벌 노코드 빌더 비교표

| 기능 | Bubble | FlutterFlow | Adalo | Glide | **앱인토스 빌더** |
|------|--------|-------------|-------|-------|-----------------|
| **출력물** | 웹앱 | 네이티브 앱 코드 | 네이티브 앱 | PWA | 토스 미니앱 |
| **백엔드** | ✅ 내장 DB | Firebase 연동 | ✅ 내장 | 스프레드시트 | ❌ 없음 |
| **로직 빌더** | ✅ 워크플로우 | ✅ FlutterFlow Logic | ✅ Actions | ✅ Actions | ❌ 없음 |
| **API 연동** | ✅ 풍부 | ✅ | 제한적 | ✅ | ❌ 없음 |
| **협업** | ✅ 팀 기능 | ✅ | ✅ | ✅ | ❌ 없음 |
| **가격** | $32+/월 | $19+/월 | $45+/월 | $25+/월 | **무료** |
| **학습 곡선** | 높음 | 중간 | 낮음 | 낮음 | **매우 낮음** |
| **타겟 유저** | 개발자/파운더 | 개발자 | 비개발자 | 비개발자 | **완전 비개발자** |

### 우리가 가진 것 (경쟁 우위)

1. **3000만 토스 유저 접근성**
   - 설치 없이 즉시 사용
   - 토스 앱 내 노출 기회
   - 결제 연동이 이미 토스 안에 있음

2. **TDS 완벽 호환**
   - 토스 네이티브 같은 UI
   - 심사 통과율 높음
   - 디자인 고민 제거

3. **심사 자동화 파이프라인**
   - 33개 검증 룰 자동 체크
   - ZIP 원클릭 다운로드
   - 5단계 제출 가이드
   - **이것만으로도 경쟁자가 없음**

4. **완전 무료**
   - 진입 장벽 0
   - 바이럴 성장 가능

5. **극도로 낮은 학습 곡선**
   - 10분 내 첫 앱 완성 가능
   - 템플릿 → 수정 → 제출 플로우

### 우리가 못 가진 것 (치명적 갭)

1. **❌ 데이터 저장소 없음**
   - 설문 응답 저장 불가
   - 주문/예약 데이터 관리 불가
   - 사용자 인증 불가
   - → 결국 "정적 페이지" 수준

2. **❌ 로직/조건부 흐름 없음**
   - 퀴즈 점수 합산 → 결과 분기 불가
   - "A 버튼 클릭 시 B 표시" 불가
   - 동적 콘텐츠 불가

3. **❌ 외부 API 연동 없음**
   - 날씨, 주식, 환율 등 실시간 데이터 불가
   - 웹훅 수신 불가

4. **❌ 토스 OAuth/유저 정보 활용 불가**
   - 개인화된 경험 제공 불가
   - 재방문 유저 인식 불가

## 🚶 2. 유저 여정 갭 분석

### 현재 퍼널

```
[랜딩] → [에디터 진입] → [컴포넌트 배치] → [제출 준비] → [ZIP 다운로드] → [콘솔 수동 업로드]
  100%      70%            40%              15%              8%                3%
```

### 이탈 포인트별 분석

#### 1단계: 랜딩 → 에디터 (30% 이탈)
- **원인**: "이게 뭐 하는 건지 모르겠다"
- **해결책**:
  - 3초 만에 이해 가능한 히어로 영상 (실제 앱 만드는 과정)
  - "1분 만에 체험하기" 인터랙티브 데모
  - 실제 심사 통과 앱 QR 코드 (토스 앱에서 바로 열림)

#### 2단계: 에디터 진입 → 컴포넌트 배치 (30% 이탈)
- **원인**: "빈 캔버스가 무서움", "뭘 해야 하는지 모르겠음"
- **해결책**:
  - 강제 온보딩 → 템플릿 선택 먼저
  - 빈 캔버스 진입 차단, 무조건 템플릿 or 가이드 퀘스트
  - "오늘의 추천 템플릿" 배너

#### 3단계: 컴포넌트 배치 → 제출 준비 (25% 이탈)
- **원인**: "내가 만든 게 제대로 된 건지 모르겠음", "이걸로 뭘 할 수 있지?"
- **해결책**:
  - 실시간 "심사 통과 가능성" 점수 표시
  - "이 앱으로 할 수 있는 것" 제안 (쇼핑몰 템플릿 → 인스타 링크 연동 등)
  - 완성 축하 모달 + "제출하러 가기" 강한 CTA

#### 4단계: 제출 준비 → ZIP 다운로드 (7% 이탈)
- **원인**: "심사가 무섭다", "개발자 아닌데 할 수 있나?"
- **해결책**:
  - 심사 체크리스트 100% 달성 시 "심사 통과 보장" 배지
  - "심사 평균 소요 시간: 3일" 명시
  - 실시간 심사 상태 알림 (이메일/푸시)

#### 5단계: ZIP 다운로드 → 콘솔 업로드 (5% 이탈)
- **원인**: "콘솔이 어디 있는지 모르겠음", "수동 업로드 귀찮음"
- **해결책**:
  - 영상 가이드 + GIF 스크린샷
  - (장기) 콘솔 자동화 API 협의 (토스와)

## 🔧 3. 기술적 부채 분석

### 즉시 해결 필요 (심각도: 🔴)

1. **pageStore와 projectStore 분리 문제**
   - `usePageStore`는 localStorage persist
   - `useProjectStore`는 Turso DB + localStorage fallback
   - 동기화 충돌 발생 가능
   - **해결**: projectStore 안에 pages를 포함하도록 통합

2. **MobileCanvasView 500ms 폴링**
   ```typescript
   const interval = setInterval(() => {
     const s = query.serialize();
     if (s && s !== "{}") setNodes(s);
   }, 500);
   ```
   - 배터리/성능 영향
   - **해결**: `query.subscribe()` 또는 Craft.js 이벤트 리스너 활용

3. **템플릿 하드코딩**
   - `templates.ts`에 935줄 하드코딩
   - 새 템플릿 추가 시 빌드 필요
   - **해결**: Turso에 템플릿 테이블 생성, 관리자 UI 추가

### 중기 개선 필요 (심각도: 🟡)

4. **컴포넌트 Props 타입 일관성 없음**
   - 일부는 `Record<string, unknown>`, 일부는 명시적 타입
   - **해결**: 모든 컴포넌트에 Props interface 정의

5. **에러 바운더리가 전체 앱을 감쌈**
   - 작은 에러에도 전체 새로고침 필요
   - **해결**: 컴포넌트 단위 에러 바운더리

6. **Craft.js resolver 수동 등록**
   ```typescript
   const resolver = {
     NavigationComponent, ButtonComponent, ...
   };
   ```
   - 새 컴포넌트 추가 시 잊기 쉬움
   - **해결**: 동적 import 또는 컴포넌트 레지스트리 패턴

### 장기 고려 (심각도: 🟢)

7. **번들 사이즈 최적화**
   - 33개 컴포넌트 전체 로드
   - **해결**: 동적 import, 컴포넌트별 코드 스플리팅

8. **테스트 코드 부재**
   - 심사 검증 로직에 버그 있으면 치명적
   - **해결**: 최소한 validation 로직에 Jest 테스트

## 💰 4. 모네타이제이션 — 사업자 없이 수익화

### 현실적 옵션

#### Option A: 프리미엄 템플릿 판매 (비추천)
- 구현: Gumroad 또는 Lemon Squeezy 연동
- 문제: 템플릿만으로 차별화 어려움, 복제 쉬움

#### Option B: "Made with Appintoss" 뱃지 제거권 (추천 ✅)
- **현재 상태**: 바이럴 뱃지 자동 삽입됨
- **수익화**: 뱃지 제거 = $5 일회성 결제
- **구현**: Lemon Squeezy 연동, 라이선스 키 발급
- **장점**: 
  - 사업자 불필요 (개인 판매자로 등록)
  - 바이럴 효과 유지하면서 수익화
  - 진짜 비즈니스 유저만 결제

#### Option C: 컨설팅/대행 (추천 ✅)
- "앱인토스 제출 대행" 서비스
- 가격: 건당 5~10만원
- 타겟: 시간 없는 소상공인, 마케터
- **구현**: 캘린더 예약 + 계좌 이체 (사업자 불필요)

#### Option D: 후원/스폰서십
- Buy Me a Coffee 또는 GitHub Sponsors
- 기대 수익: 월 10~30만원
- 바이럴 시 괜찮은 부수입

#### Option E: 교육 콘텐츠 (장기)
- "노코드로 토스 미니앱 만들기" 온라인 강의
- Class101, 인프런 등
- 예상 수익: 강의당 100~300만원


## ⚔️ 7. 킬러 기능 — 경쟁자가 못 따라오는 무기

### 현재 킬러 기능

#### 🎯 "심사 통과 보장" 시스템
- 경쟁자: 없음
- 이유: 앱인토스 전용 도구라 시장 자체가 없음
- 강화 방안:
  1. 심사 통과율 실시간 표시 (현재 프로젝트: 87%)
  2. "체크리스트 100% = 통과 보장" 마케팅
  3. 심사 탈락 시 무료 컨설팅 (신뢰 구축)

### 개발해야 할 킬러 기능

#### 🧠 "AI 앱 생성" (3개월 내)
```
입력: "우리 카페 포인트 적립 앱 만들어줘"
출력: 완성된 앱 (로고, 색상, 내용 커스텀)
```
- 구현 방안: 
  - Claude API로 JSON 스키마 생성
  - 프롬프트: "토스 미니앱 JSON 생성, 컴포넌트: [목록]"
  - 예상 비용: API 호출당 0.01~0.05달러
- 차별점:
